About This Repository

`kschema-fs-api-gen-get-actions` is a core utility designed to parse configurations, load modular JS templates dynamically, and output production-ready boilerplate code.

⚡

Performance

Scaffolds full-stack endpoint routes, validators, data access layers (DALs), and controller maps in milliseconds.

🧩

Dynamic Architecture

Configured using `actions.json`. Simply drop in a new module configuration and it is immediately available in the CLI without hardcoding.

🔒

Safe Outputs

Includes built-in validation checks protecting files from unintended duplication and duplicate imports.

Quick Start

Run the package globally or trigger it instantly using npx.

bash
npm install -g @keshavsoft/kschema-api-gen-actions # Run the dynamic action generator command npx @keshavsoft/kschema-api-gen-actions ShowAll

Internal Architecture Flow

How the JSON-driven dynamic command loading operates inside the CLI core.

1
CLI Parser parses arguments
↓
2
resolveCommand.js references actions.json
↓
3
loadCommand() runs dynamic import()
↓
4
Scaffolding engine writes templates

Automated CI/CD Actions

Continuous Deployment & automatic dependency updates configured across our cascading repositories.

🚀

Auto Publish & Notify

Whenever a release is published, the workflow automatically publishes to npmjs. Upon success, it dispatches notifications to downstream dependants.

🛡️

Duplicate Protection

The publishing action checks if the current package version already exists on NPM before triggering `npm publish`. This ensures workflows don't fail if the package is already updated.

⛓️

Propagation chain

Automatically triggers `vs-code-ext-express-api-gen-get-actions` to run its dependency pull, making sure developers always use the latest features.

Supported Commands
