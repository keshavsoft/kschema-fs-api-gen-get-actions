Update Dependency Action

Details of the dynamic inbound update workflow (`update-dependency.yml`).

Workflow Trigger

The `update-dependency.yml` workflow listens to `repository_dispatch` events sent from upstream repositories (specifically, express-fix-endpoints-get-js) with the event type dependency-updated:

yaml
on: repository_dispatch: types: [dependency-updated] workflow_dispatch:

Step 1: Install Latest Upstream Dependency

The action runs npm installation commands to grab the updated upstream dependency package version:

yaml
- run: npm install - run: npm install express-fix-endpoints-get-js@latest

Step 2: Version Patch Increment

If package changes are detected by the update, a patch version is bumped:

bash
git diff --quiet package.json || npm version patch --no-git-tag-version

We use --no-git-tag-version to prevent git tag creation from triggering redundant deployment runs.

Step 3: Commit and Push Changes

The local updates are pushed back to the main branch using git details of the GitHub Actions Bot:

bash
- name: Commit and push changes run: | git config --global user.name "github-actions[bot]" git config --global user.email "github-actions[bot]@users.noreply.github.com" git add package.json package-lock.json if ! git diff --cached --quiet; then git commit -m "Update express-fix-endpoints-get-js to latest and bump version" git push else echo "No changes to commit" fi
