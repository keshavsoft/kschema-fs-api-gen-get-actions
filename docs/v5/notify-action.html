Notify Dependents Action

Details of the downstream repository dispatch notification webhook.

Post-Publish Notifications

Once publishing is verified, we need to notify downstream projects. In `kschema-fs-api-gen-get-actions`, the dependents are notified via the reusable workflow notify-dependents.yml.

Step 1: Wait for NPM CDNs

To avoid race conditions where downstream workflows run before NPM finishes replicating the new build version globally, we sleep the runner for 15 seconds:

yaml
- name: Wait for NPM registry replication run: sleep 15

Step 2: API Repository Dispatch Webhook

The workflow calls GitHub's repository dispatch API, triggering the `dependency-updated` event inside the downstream project vs-code-ext-express-api-gen-get-actions:

bash
curl -i -X POST \ -H "Accept: application/vnd.github+json" \ -H "Authorization: Bearer ${{ secrets.REPO_DISPATCH_TOKEN }}" \ https://api.github.com/repos/keshavsoft/vs-code-ext-express-api-gen-get-actions/dispatches \ -d '{"event_type":"dependency-updated"}'

Parameters Breakdown:

  • Authorization: Bearer header authorized with a Personal Access Token (`REPO_DISPATCH_TOKEN`) with write permissions.
  • Payload Type: `event_type` is set to `dependency-updated` so the downstream project filters the trigger.
