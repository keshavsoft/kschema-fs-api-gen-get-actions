CI/CD Actions

Automatic dependency propagation chain and publishing workflows.

Cascading Update Pipeline

We use GitHub Actions dispatches to propagate updates downstream instantly whenever a base component changes version:

express-check-any-for-import publishes to NPM
↓ dispatch
express-fix-any-js pulls latest dependency & bumps version
↓ dispatch
express-fix-endpoints-get-js pulls latest dependency & bumps version
↓ dispatch
kschema-fs-api-gen-get-actions pulls latest dependency & bumps version
↓ dispatch
vs-code-ext-express-api-gen-get-actions pulls latest dependency

NPM Publish Safety Check

To prevent workflows from failing when versions haven't changed, the `npm publish` command dynamically checks the NPM registry first:

yaml
- name: Publish to npm run: | PACKAGE_NAME=$(node -p "require('./package.json').name") PACKAGE_VERSION=$(node -p "require('./package.json').version") if npm view "${PACKAGE_NAME}@${PACKAGE_VERSION}" version >/dev/null 2>&1; then echo "Version ${PACKAGE_VERSION} of ${PACKAGE_NAME} is already published on npm. Skipping publish." else npm publish fi env: NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}

Required Secrets

To run these workflows successfully, define the following variables under Settings -> Secrets and Variables -> Actions:

Secret Name Description
NPM_TOKEN An access token from npmjs.org configured with write access to deploy package packages.
REPO_DISPATCH_TOKEN A GitHub Personal Access Token (PAT) with `repo` read/write scopes to execute cross-repository dispatch triggers and push version commits back to main branches.
