NPM Publish Action

Analysis of the package publishing pipeline and safety-checking mechanisms.

Workflow Triggers

The publishing process in publish-and-notify.yml is triggered by:

  • Release Publish: Executed when a release is transitioned to the "published" state (release: types [published]).
  • Manual Run: Triggered manually via the Actions UI tab (workflow_dispatch).

NPM Publish Safety Check

To avoid build failures when attempting to deploy a version that is already registered on the registry, a pre-publish script queries the NPM database first:

yaml
- name: Publish to npm run: | PACKAGE_NAME=$(node -p "require('./package.json').name") PACKAGE_VERSION=$(node -p "require('./package.json').version") if npm view "${PACKAGE_NAME}@${PACKAGE_VERSION}" version >/dev/null 2>&1; then echo "Version ${PACKAGE_VERSION} of ${PACKAGE_NAME} is already published on npm. Skipping publish." else npm publish fi env: NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}

Step Breakdown:

  • PACKAGE_NAME and PACKAGE_VERSION are dynamically retrieved from package.json using Node.js's print execution mode (`node -p`).
  • npm view ... checks if the package version is listed on npmjs.org. If yes, it skips the step successfully. If not, it executes `npm publish`.
